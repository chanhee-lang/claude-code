### 1. MCP 설정은 settings.json이 아니라 .claude.json에 저장된다

`settings.json`에는 `mcpServers` 필드가 없다. MCP 서버 설정의 실제 저장 위치는 `~/.claude.json`이다.
`settings.json`은 Claude Code의 동작 설정(권한, 훅 등)을 담고, MCP 서버 목록은 `.claude.json`이 담당한다.
파일 직접 수정 대신 CLI를 사용해야 올바른 파일에 올바른 형식으로 저장된다.

```bash
# 올바른 MCP 추가 방법
claude mcp add <server-name> <command>

# 현재 등록된 MCP 확인
claude mcp list
```

### 2. Playwright MCP로 Claude가 직접 브라우저를 조작할 수 있다

`@playwright/mcp@latest`를 추가하면 Claude 세션 안에서 브라우저를 열고, 클릭하고, 스크린샷을 찍는 것이 가능해진다.
E2E 테스트 자동화, UI 검증, 웹 스크래핑을 별도 스크립트 없이 대화 흐름에서 처리할 수 있다.

```bash
claude mcp add playwright npx @playwright/mcp@latest
```

### 3. 홈 디렉토리(~/)에서 git init하면 안 된다

`~/`에서 `git init`을 실행하면 `.ssh`, `.aws`, `.env`, `.claude.json` 등 민감한 파일 전체가 추적 대상이 된다.
설정 파일 버전 관리가 필요하면 반드시 별도 디렉토리를 만들어서 관리해야 한다.

```bash
# 잘못된 방법
cd ~/ && git init   # 절대 하지 말 것

# 올바른 방법
mkdir ~/claude-code && cd ~/claude-code && git init
```

### 4. [BLOCK] 시크릿(토큰, 비밀번호)을 채팅에 직접 붙여넣으면 안 된다

Slack Bot Token이 채팅 내용에 노출됐다. 채팅 로그는 히스토리로 남고, 어디에서 참조될지 모른다.
노출된 즉시 토큰을 폐기하지 않으면 공격자가 해당 토큰으로 API를 호출할 수 있다.

**노출 시 즉시 조치 순서:**
1. 해당 서비스 콘솔에서 토큰 즉시 revoke
2. 새 토큰 재발급
3. 로컬 환경에 `.env` 파일로 저장

```bash
# .env 파일로 관리
SLACK_BOT_TOKEN=xoxb-your-token-here

# .gitignore에 반드시 추가
echo ".env" >> .gitignore
```

## 실수 & 반성

- MCP 설정 파일 위치를 추측해서 직접 수정하려 했다. CLI 명령을 먼저 확인했어야 했다.
- 시크릿을 채팅에 붙여넣었다. 설정값은 항상 파일로 전달하거나, 환경 변수 이름만 언급하는 습관을 들여야 한다.

## 새로운 발견

- Claude Code의 설정 파일 구조: `settings.json`(동작 설정) vs `~/.claude.json`(MCP 서버 목록)은 역할이 분리되어 있다.
- Playwright MCP는 기존 브라우저 자동화 스크립트를 대체할 수 있는 강력한 도구다. 별도 테스트 프레임워크 세팅 없이 바로 활용 가능하다.
